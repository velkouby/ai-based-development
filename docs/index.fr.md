# Faire travailler les agents de code dans un système vérifiable

Des principes et des pratiques concrètes pour passer d'une conversation avec un agent à un développement explicite, contrôlé et relisible.

La série suit un même fil : préparer le repository, choisir un niveau de contrôle proportionné au risque, compiler le contexte d'exécution, savoir quand arrêter puis reprendre le travail et produire des preuves dont la portée est claire.
